◆ bhmSerOpen()

bhmChannelHandle bhmSerOpen ( Uns32 *  portp,
const char *  logfile,

Bool  client,
Bool  nonblocking,
Bool  verbose,
Bool  console,
Bool  finishOnDisconnect,
Uns32  xchars,
Uns32  ychars 
)

Create a new serial channel, blocking the simulator until it its connected to an external program

Parameters
portpPointer to integer port number. If null, sockets are not used. If pointing to zero, a port is allocated by the system and the value overwritten. If non-zero, the port number is used. Ignored if console is true, overrides parameter sourcefile
logfileA string specifying a file to record all serial device output.
outfileA string specifying a file to record the raw serial output
sourcefileA file to use as source for the serial device. Note that this data will be read as fast as the model requests it. Ignored if console is true. Ignored if portp is non-null.
portfileUsed in conjunction with portp. A file with this name is written when the port number is known. Overridden by the console parameter.
clientIf true, model is a client and will connect to portnum. If false, model is a server and will listen on portnum.
nonblockingIf set, simulator will not wait for a connection. Simulation will begin before the connection is made. If false, model will wait until a connection is made
verboseSend UART output to the simulator log
consolePop up a console window, and connect using a socket. Overrides the port and sourcefile parameters.
The terminal program which creates the console is $IMPERAS_HOME/bin/[architecture]/terminal.exe which is supplied with Product and Demo packages. The program can be overridden to use a specified program or script using the environment variable IMPERAS_TERMINAL_COMMAND, e.g.
On Linux
export IMPERAS_TERMINAL_COMMAND=/usr/bin/telnet
export IMPERAS_TERMINAL_COMMAND=$(pwd)/startTerminal.bash
On Windows
set IMPERAS_TERMINAL_COMMAND = C:\startTerminal.bat
The specified program or script will be passed two arguments, the system host name and the port number that was opened.
On Linux the xterm program is used to launch the terminal program. If this is in a non-standard location it can can be specified using the environment variable IMPERAS_XTERM_PATH, e.g.
export IMPERAS_XTERM_PATH=/home/user/xterm
finishOnDisconnectWhen the socket disconnects, finish the simulation.
xcharsIf in console mode, specify the width of the console in characters,
ycharsIf in console mode, specify the height of the console in characters,
Returns
The channel number, or BHM_INVALID_HANDLE if it fails

bhmChannelHandle bhmSerOpenAuto ( void  )

outfile string Name of file to write device output (can be used in addition to other modes).

Create a new serial channel, blocking the simulator until it its connected to an external program. Derive all the operating modes of the serial channel from model parameters instead of arguments. These are the parameters: console string If specified, port number is ignored, and a console pops up automatically. This source is the 1st priority input. portnum integer If set, listen on this port.
