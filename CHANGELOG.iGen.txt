TLM platform
-------------------------------------------------------------------------------
-iGen now writes TLM platforms that do not include model-specific interface
  files.
  Each instance of a processor, MMC or peripheral had an interface file
  included in the sc_constructor.h. This is no longer used; the platform
  includes the generic processor, peripheral and MMC model interface files.
  This means that the interface files do not change when model parameter values
  change the number of names of bus or net ports on the model.
  
  The new platforms are functionally identical to the old.
  
  To revert to iGen's old behaviour, add -wrappers to the iGen command line
  (and generate any required interface files).
  
  In future, Imperas will no longer ship tlm model interface files in a
  standard release, but interface files will be available on request.
  
- iGen now writes calls to opVLNVString using the new enumerated type in the
  last parameter.

      Address calculations in the macros.igen.h file
      Memory mapped register write-mask
  iGen now writes a check for address misalignment in its callbacks.
  To the command imodeladdbusslaveport -remappable 
  the -addresswidth option is now mandatory. It represents the number of
  address bits used by the remappable (dynamic) slave port.
  
- A SystemC Imperas model interface from iGen now
  - is generated by loading the pse.pse model file, instead of the TCL file.
  - correctly detects if the model is not available and shuts down correctly.

- When a platform or module passes a parameter to an instance, the type is 
  now checked for compatibility. Previously, an error in the TCL could result
  in incorrect C code being generated.
  
- Processor bus slave ports
  Some processor models now have bus slave ports. Connecting a processor bus
  slave port in a module or platform would cause a SEGV. This has been fixed.

- Wide registers
  The tcl command imodeladdmmregister -nbyte
  which should force igen to use ppmCreateNByteRegister()
  was sometimes ignoring the flag. This is now fixed.
  
- Peripheral Macro File
  Additional macros are generated into pse.macros.igen.h to aid the extraction
  of bit fields from peripheral registers in application code.

- SystemC
  iGen now generates references to the reorganized Imperas SystemC TLM 
  interface classes (see CHANGELOG.OVP.txt).
  
- Peripheral address block arrays
  can now be created using    imodeladdaddressblock -loarray <n> -hiarray <m>
  If required, any wide register (using -nbyte) can be passed the userdata value
  _addressblockindex which is the array index:
           imodeladdmmregister -userdata _addressblockindex

igen TCL
-------------------------------------------------------------------------------
- A new flag -nbyteregisters is accepted by imodelnewperipheral.
  Setting this causes iGen to use ppmCreateNByteRegister() for all registers
  and ppmInstallNByteCallbacks() for local memories.

- A new flag -nbyte is accepted by imodeladdmmregister.
  Setting this causes iGen to use ppmCreateNByteRegister() for this register.
  
- A new flag -nbyte is accepted by imodeladdlocalmemory.
  Setting this causes iGen to use ppmInstallNByteCallbacks() for this memory.

- A new flag -nonvolatile is accepted by imodeladdlocalmemory. Setting this
  prevents a write callback when the written value is the same as the last
  write.

- A new flag -formalvalues is accepted by imodelnewperipheral.
  Setting this causes iGen to create a variable for each formal parameter
  (created using imodeladdformal) and to generate code to set this variable
  to the value of the parameter. The value is set using the new field valuePtr
  in the ppmParameter. 
  
  - A new flag -reset <integer> is accepted by imodeladdfield.
  The integer is the value taken by the field when the register is reset.
  A register can use the  -mask <value> and -reset <value> which applies to
  the who register or can use a value per field (which does not need a mask).
  Register and field values cannot be mixed. If a reset net specified, a value
  must be supplied to the register or to at least one field.
  For registers larger than 64 bits, reset values must be applied to the
  field, not the register. If a field is larger than 64 bits it must be split
  into smaller fields.
  
- The flag -autoload to command imodelnewsemihostlibrary now reports that it
  is not required (It has been unused for some time).
  
- A new flag -defaultfunctions is accepted by imodeladdbusslaveport.
  Setting this causes iGen to install read and write callbacks on the entire
  slave port address region. The callbacks will report any access that does
  not address a memory mapped register or a local memory.
  
- Clarification. The command imodeladdsupportedprocessor is used during in
  document generation but has no effect on the intercept library being created.
  
- Introduced support for peripheral formal parameters of type enum.

igen TCL syntax
-------------------------------------------------------------------------------
- TCL command imodeladdformal.
  The flag -defaultValue was defined with a capital V which is inconsistent
  with other flags. It is now named -defaultvalue. igen now ignores the case of
  flag names so TCL code does not need to be changed.

New -netportuserfunction option added to imodelnewperipheral command
-------------------------------------------------------------------------------
- A new option named -netportuserfunction has been added to the
  imodelnewperipheral command. This specifies a user defined function to be 
  called when the simulator is interrogating the peripheral model about the 
  netports. This allows the user to define additional netports beyond those that
  iGen creates, e.g. when the number of netports is determined by the value of a  
  runtime parameter.
  
- The endian parameter is now correctly written in a TLM processor instance.

- iGen now creates calls to ppmCreateRegisterField so that documentation
  can be produced for register bit-fields.

- OP modules that use formal parameters.
  Certain values in an OP module can be set by the value of a parameter
  declared in that model. In the TCL simply refer to the parameter name in place
  of a constant value. Values that can be parameterized now include:
       - a parameter value in a module or component instance.
       - a VLNV vendor,library,name or version
       - a model path
       - a bus,net,fifo,packetnet port name on a module or component instance.
       - an instance name
       - path to a loaded program
       - path to a processor extension, intercept or plugin

- Creating a peripheral model bus master port
  The igen command imodeladdbusmasterport now accepts extra options
  -addresswidthmin and -addresswidthmax to specify the allowed range of address
   bits on the bus that connects to the port.
      
- Creating a dynamic port
  When the slave port is dynamic i.e. using the -remappable argument the slave 
  port pointer must still be generated in the handles structure.
  A port no longer requires to have low and high addresses defined in order to 
  add registers into it, so dynamic ports can now be supported.

- Creation of OP modules has been enhanced
  When creating modules, iGen now creates the following:
  
  module.igen.h file
  ------------------
  // instantiate module components
  static OP_CONSTRUCT_FN(instantiateComponents) {
      . . . OP module component instantiation . . .
  }
    
  module.c.igen.stubs file
  ------------------------
  // forward declaration of component constructor
  static OP_CONSTRUCT_FN(instantiateComponents);

  static OP_CONSTRUCT_FN(moduleConstructor) {

      // instantiate module components
      instantiateComponents(mi, object);

      // insert constructor code here
  }

  This enhanced structure enables user code to be added to moduleConstructor
  which is called in the construction phase. Previously, the earliest that user
  code code get control was in the pre-simulation phase.
  
  - Changing the protection of a region
  A new tcl command changes the protection of a region: 
     ihwaddprotection  \
         -bus <bus>    \
         -loaddress <addr> \
         -hiaddress <addr> \
         -privilege <priv>

iGen
-------------------------------------------------------------------------------
- Using -op mode, iGen now writes the new version of the OP API
  (see CHANGELOG.OP.txt).
  
- Added a new command:
    ihwaddnet -instancename <net name> 
    
  Previously, a net was implicitly created when referred to in a connection.
  When writing an OP platform or module, each net must be created explicitly.
  When writing any platform, if ihwaddnet is used at all, then ALL nets must be
  created explicitly.
  
- isetattribute can set create OP code set a parameter to a value which is a
  C expression rather than a constant. Use -expression instead of -value:
    isetattribute -name <name> -type <type> -expression <C expression>
      
- most ihw commands that accept integer valued arguments can now accept 
  C expressions where constants were accepted before. e.g:
    ihwaddformal -name base -type address 
    ihwconnect -bus b -instancename u1 -busport p \
        -loaddress {base+100} \
        -hiaddress {base+200} 
      
- igen no longer inserts a default value for:
    ihwaddbus -addresswidth      this value is always required.
    ihwaddprocessor -mips        if a value is required, it must be specified.

iGen
-------------------------------------------------------------------------------
- NOTE The function of -writec has changed:

  igen.exe -batch <source>.tcl -writec <directory>/<file>[.ext]
  
  Files are created in <directory>
  Each file begins with <file>
  Files that do not need to be modified are called <file>.igen.<something>
  
  The file that can modified has a special rule:
  If [.ext] is absent .igen.stubs is appended, else the given name is used. 
  
  e.g. For a peripheral model
  igen.exe -batch peripheral.tcl -writec peripheral
  creates
      peripheral.igen.stubs
      peripheral.igen.c
      peripheral.igen.h
      peripheral.attrs.igen.c
      peripheral.macros.igen.h
      
  igen.exe -batch peripheral.tcl -writec peripheral.c
  creates
      peripheral.c
      peripheral.igen.c
      peripheral.igen.h
      peripheral.attrs.igen.c
      peripheral.macros.igen.h

- Use the new command line flag -newargparser to make igen create a template
  for an intercept library that uses vmirt calls to parse model commands. 
        
- Fixed bug where the -semihost* parameters on ihwaddprocessor commands were
  ignored when generating a TLM platform.
  
- PSE C port and parameter iterators are now correct for an empty list.

- igen writes a TLM platform without using model xml files, but the executable
  models must be in the library.
  
- Like the simulator, igen now uses the environment variable IMPERAS_VLNV
  as well as the command line argument --vlnvroot.
  
- Removed igen interactive commands to manipulate libraries, and to write
  output files. Igen can be used interactively to experiment with commands
  and their arguments, but must be used in batch mode to produce output.
  
  Deleted commands:
  iclear                   : Remove model or platform currently being built.
  ifind                    : Search for all items matching a pattern.
  iget                     : Get object information from a handle.
  ihwfree                  : Finish editing the current hardware design.
  ihwsave                  : Save a hardware design.
  ihwshow                  : Pretty print the hardware design being edited.
  ihwverify                : Verify a hardware design.
  ilibrarypath             : Perform operations on the library read path.
  ilibraryreload           : Reload the library cache.
  ilibrarysavepath         : Set and get the library save path.
  ilibraryshow             : Summary of the library cache objects.
  ilibraryshowmodel        : Show the details of a model.
  imodelfree               : Finish editing the current model.
  imodelgenerateperipheral : Create a template for a peripheral model.
  imodelgenerateplatform   : Create a template for a platform.
  imodelgeneratetlm        : Create a TLM interface.
  imodelsave               : Save the current model.
  imodelshow               : Pretty print the model being edited.

- Improved error reporting for registers created with overlapping addresses.

- Removed the flag --nouserinit  - it is no longer required.
- Added the flag   --checkmodels - When writing a C platform, igen will load
  each model and verify its interface against the platform TCL.

iGen
-------------------------------------------------------------------------------
- An unnecessary write mask check for read functions has been removed. 
  Different registers may now specify the same read function even when they
  have different write masks defined.
- igen does not incorrectly generate an unused register view function.
- igen no longer generates a ppmCreateSlaveBusPort() call for slave ports with
  a size of 0. The user-defined code is responsible for instantiating the 
  port in this case.
- The writec argument for platform creation has been changed so that it is now
  provided with a stub name for the output files, rather than the user file name. 
  The VLNV name in the tcl source file is no longer used.
  As an example, the argument to igen.exe of the form
      --writec my/path/to/file/stubName 
  will generate a set of files
      my/path/to/file/stubName.c.igen.stubs
      my/path/to/file/stubName.clp.igen.h
      my/path/to/file/stubName.constructor.igen.h
      my/path/to/file/stubName.handles.igen.h
      my/path/to/file/stubName.options.igen.h
  where the file my/path/to/file/stubName.c.igen.stubs is moved to a C file 
  stubName.c and modified by the user for platform initialization or argument
  parsing.
  See the iGen User manual for further information.
- When writing formal parameters for an intercept library, igen now uses macros
  defined in vmiParameters.h

- Writing peripheral registers
  When writing a peripheral model, the register name passed to
  ppmCreateRegister, now includes the name of the containing address block,
  to prevent name clashes in more complex models. This changes the name of 
  the register and of its derived view events.

  name for the output files, rather than the user file name. The VLNV name in 
  the tcl source file is no longer used.
  As an example, the argument to igen.exe of the form
      --writec my/path/to/file/stubName 
  will generate a set of files
      my/path/to/file/stubName.attrs.igen.c
      my/path/to/file/stubName.igen.c
      my/path/to/file/stubName.igen.h
      my/path/to/file/stubName.igen.stubs
      my/path/to/file/stubName.macros.igen.h
  where the file my/path/to/file/stubName.igen.stubs is moved to a C file and
  modified by the user.
  See the iGen User manual for further information.

User Build Infrastructure
-------------------------------------------------------------------------------
- The generation of peripheral model source code by Makefile.igen has been 
  updated to use the (recommended) argument, '-writec'. This may change the 
  generated filenames. If the source file is pse.tcl there should be no change.
- TCL Source file naming, no longer supports TCL source files with a specific 
  name, for example  <name>.pse.tcl, <name>.platform.tcl etc.
  TCL source files must have a specific name to be found by the build system
  Peripheral :  pse.tcl
  Platform   :  platform.tcl
- The PSE Makefile, Makefile.pse, has been enhanced to support the generation of
  peripheral source files from the tcl file, if a file of type pse.tcl exists 
  in the local directory.

Changes to IGEN TCL commands
-------------------------------------------------------------------------------
- The argument -attributetable is no longer required so has been deleted from
  commands imodelnewprocessor, imodelnewmmc & imodelnewsemihostlibrary
    
Enhancement to IGEN 
-------------------------------------------------------------------------------
- igen.exe output files fall into 2 categories;
       1. files that need to be modified
       2. files that do not normally need to be edited.
  Files in category 2. will not be overwritten by igen.exe unless
        --overwrite is specified
        
- command line arguments to be deprecated:
--writecattrs            Use -writec
--writecinclude          Use -writec
--writecmacros           Use -writec
--writecmodel            Use -writec
--writecosobj            Use -writec
--writecplatform         Use -writec
--writecstubs            Use -writec
 
Enhancement to IGEN generated TLM Platform
-------------------------------------------------------------------------------
- The generated platform can now include a command line parser (CLP).
  Use the TCL command ihwaddclp to include the CLP.
  Use the TCL command ihwaddclparg to add a custom argument to the CLP

- The platform is now written as 4 or 5 files:
    <output>.c              specified by -writectlm <output>.c
                            contains the platform constructor and main()
    <name>.sc_options.igen.h   from the platform name
                            contains variables set by the CLP.
    <name>.sc_clp.igen.h       from the platform name, if requested
                            contains the command line parser.
    <name>.sc_functions.igen.h from the platform name
                            contains stub functions to be modified by the user.
                            
    (all files are put in the same directory)
                            
- <output>.c now defines sc_main
                                
Enhancement to IGEN generated ICM Platform
-------------------------------------------------------------------------------
- The generated platform can now include a command line parser (CLP).
  Use the TCL command ihwaddclp to include the CLP.
  Use the TCL command ihwaddclparg to add a custom argument to the CLP

- The platform is now written as 4 or 5 files:
    <output>.c              specified by -writec <output>.c
                            contains the platform constructor and main()
    <name>.options.igen.h   from the platform name
                            contains variables set by the CLP.
    <name>.handles.igen.h   from the platform name
                            contains handles for created objects.
    <name>.clp.igen.h       from the platform name, if requested
                            contains the command line parser.
    <name>.functions.igen.h from the platform name
                            contains stub functions to be modified by the user.
                            
    (all files are put in the same directory)
                            
The 1st file includes the others so the output is compatible with the previous
release.

- It is now possible to write an attribute with numeric value > 32 bits.

Enhancement to IGEN generated Peripherals and Plugins
-------------------------------------------------------------------------------
- When writing template files for peripherals and plugins 
  the single argument -writec <filename> replaces the multiple arguments
  previously required.
                                
Enhancement to IGEN generated SystemC TLM Processor interfaces
-------------------------------------------------------------------------------
The following additional optional arguments have been added to the processor  
class constructors generated by igen for a SystemC TLM processor interface:

        Uns32 addressBits (defaults to 32)
        bool  dmi         (defaults to true)
        Uns32 cpuFLags    (defaults to 0)

Previously these values could not be set from an instantiation of a processor
in a SystemC TLM platform using the igen-generated class definition. Now the 
user may specify the values to be used. (These have default values defined for 
them, so they are optional arguments, so no changes to existing platforms should 
be necessary.)
